# เกี่ยวกับการเปรียบเทียบ branches ใน pull requests

> เอกสารอ้างอิง: GitHub Docs (แปลเป็นภาษาไทยโดย AI)

Pull requests จะแสดง diffs เพื่อเปรียบเทียบการเปลี่ยนแปลงที่คุณทำใน topic branch กับ base branch ที่คุณต้องการจะ merge การเปลี่ยนแปลงของคุณเข้าไป

> [!NOTE]
> เมื่อคุณสร้าง pull request คุณสามารถเปลี่ยน base branch ที่ใช้เปรียบเทียบการเปลี่ยนแปลงของคุณได้
> ดูเพิ่มเติม: [การสร้าง pull request](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request)

## การดูการเปลี่ยนแปลง

คุณสามารถดูการเปลี่ยนแปลงที่เสนอใน pull request ได้ในแท็บ **Files changed**

- ไฟล์ต่างๆ แสดงตามลำดับตัวอักษร
- ส่วนที่เพิ่ม (+) → สีเขียว
- ส่วนที่ลบ (-) → สีแดง

> [!TIP]
> หากเข้าใจบริบทได้ยาก คลิก **View** เพื่อดูไฟล์ทั้งหมด

## ตัวเลือกการแสดงผล diff

| ตัวเลือก | คำอธิบาย |
|----------|----------|
| **Unified view** | แสดงแบบเส้นตรง (ใหม่+เก่าด้วยกัน) |
| **Split view** | แยกซ้าย (เก่า) / ขวา (ใหม่) |
| **Rich diff** | แสดงตัวอย่างหลัง merge |
| **Source view** | แสดงซอร์สโค้ดล้วน |

**เพิ่มเติม:**
- ✅ สามารถไม่สนใจ whitespace ได้
- ✅ กรองเฉพาะไฟล์ที่ต้องการดูได้
- ✅ ดู dependency changes และ security vulnerabilities ได้

---

## สาเหตุที่ diff ไม่แสดง

1. **เกินขีดจำกัด** — จำนวนไฟล์หรือประเภทไฟล์เกิน limit
2. **ถูก .gitattributes บล็อก** — ไฟล์ถูกกำหนดให้ไม่แสดง

## Git diff แบบสองจุด vs สามจุด

### การเปรียบเทียบ

| แบบ | คำสั่ง | ความหมาย |
|-----|--------|----------|
| **สองจุด (two-dot)** | `git diff A..B` | เปรียบเทียบ A กับ B โดยตรง |
| **สามจุด (three-dot)** | `git diff A...B` | เปรียบเทียบ merge base กับ B |
