# Crystal Castle – Markdown Style Guide

## 🔎 Purpose
กำหนดมาตรฐานการเขียน Markdown สำหรับเอกสารใน Crystal Castle  
เพื่อให้ทีมใหม่ onboard ได้ง่าย และ reviewer ตรวจสอบได้เร็ว

---

## 📝 Headings
- ใช้ `#` สำหรับหัวข้อใหญ่ (H1) → ชื่อเอกสาร
- ใช้ `##` สำหรับหัวข้อรอง (H2) → section หลัก
- ใช้ `###` สำหรับหัวข้อย่อย (H3) → subsection
- ห้ามใช้เกิน H3 เพื่อความอ่านง่าย

---

## ✨ Emphasis
- ตัวหนา: `**ข้อความสำคัญ**`
- ตัวเอียง: `*คำอธิบายเพิ่มเติม*`
- หลีกเลี่ยงการใช้ตัวหนา/เอียงซ้ำซ้อน

---

## 📋 Lists
- Bullet list: `- รายการ`
- Numbered list: `1. รายการ`
- Checklist: `- [ ] งานที่ต้องทำ` / `- [x] งานที่เสร็จแล้ว`

---

## 📊 Tables
- ใช้ `|` และ `---` กำหนดตาราง
- ตัวอย่าง:
  ```markdown
  | วันที่   | รายการ   | จำนวน |
  | -------- | -------- | ----- |
  | 01/01    | ค่าอาหาร | 60    |
  | 02/01    | ค่าเดินทาง | 50   |
