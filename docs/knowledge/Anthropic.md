Anthropic ได้เปิดตัว **"Routines"** ใน Claude Code ซึ่งเป็นการอัปเกรดครั้งสำคัญจากการสั่งงานรายครั้ง (Interactive sessions) ไปสู่การทำงานแบบ **อัตโนมัติ (Automation)** อย่างเต็มรูปแบบ
สรุปประเด็นสำคัญจากประกาศมีดังนี้ครับ:
### 1. Routines คืออะไร?
Routines คือการบันทึกการตั้งค่าของ Claude Code (ประกอบด้วย คำสั่ง/Prompt, คลังเก็บโค้ด/Repositories และเครื่องมือเชื่อมต่อ/Connectors) เพื่อให้ทำงานโดยอัตโนมัติเมื่อถึงเวลาหรือมีเหตุการณ์ที่กำหนด โดยไม่จำเป็นต้องเปิดคอมพิวเตอร์ทิ้งไว้
### 2. จุดเด่นที่สำคัญ
 * **Cloud Execution:** ทำงานบนโครงสร้างพื้นฐานของ Anthropic โดยตรง (Cloud-native) ต่างจากระบบ /schedule เดิมที่ต้องเปิดแอป Desktop ทิ้งไว้เพื่อให้งานเริ่มทำงาน
 * **Triggers (ตัวจุดชนวน):** รองรับ 3 รูปแบบหลัก
   * **Scheduled:** ตั้งเวลาล่วงหน้า (รายชั่วโมง, รายวัน, รายสัปดาห์)
   * **API:** เรียกใช้งานผ่าน HTTP endpoint เพื่อเชื่อมต่อกับระบบอื่น
   * **Webhooks:** สั่งงานตามเหตุการณ์ เช่น เมื่อมีการสร้าง Pull Request ใหม่บน GitHub
 * **Autonomy:** ทำงานแบบอิสระโดยไม่ต้องรอการอนุมัติ (Permission-free) ในระหว่างที่โปรเซสกำลังรันอยู่ จึงเหมาะกับงานที่ต้องการความต่อเนื่อง
### 3. ตัวอย่างการใช้งาน (Use Cases)
 * **Code Maintenance:** ตรวจสอบช่องโหว่ความปลอดภัยรายสัปดาห์ หรืออัปเดต Dependencies อัตโนมัติ
 * **Daily Triage:** ให้ Claude สรุป Issues ใหม่ๆ ที่เกิดขึ้นใน GitHub ทุกเช้า
 * **CI/CD Enhancement:** ทำการตรวจโค้ด (Code Review) เบื้องต้นทันทีที่มีคนส่ง Pull Request
 * **Content Ops:** ดึงข้อมูลจากฐานความรู้มาเขียนร่าง Blog หรือเอกสารทางเทคนิคตามรอบเวลา
### 4. การเข้าถึงและการใช้งาน
 * **รองรับทุกแพลตฟอร์ม:** จัดการได้ทั้งผ่านหน้าเว็บ (claude.ai/code/routines), ผ่าน CLI ด้วยคำสั่ง /schedule, หรือผ่านแอป Claude Desktop
 * **สิทธิ์การใช้งาน:** พร้อมใช้งานสำหรับผู้ใช้แผน Pro, Max, Team และ Enterprise
 * **ข้อควรระวัง:** เนื่องจากการทำงานเป็นแบบอัตโนมัติ 100% (Unsupervised) ผู้ใช้จำเป็นต้องระบุความคาดหวังและขอบเขตงานใน Prompt ให้ชัดเจนที่สุด เพื่อป้องกันความผิดพลาดครับ
