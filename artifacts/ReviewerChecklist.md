# ✅ Reviewer Checklist - Artifacts Folder

This section ensures that files in `artifacts/` follow project policy.

## 📌 Checks (English)
- [ ] Confirm all files in `artifacts/` are outputs from workflows or manual runs only.
- [ ] Verify that no production source code is stored in `artifacts/`.
- [ ] Ensure file names are clear and descriptive (e.g., `ReviewerPolicy.pdf`, `test-report-YYYYMMDD.log`).
- [ ] Confirm artifacts are temporary and not mixed with source code.
- [ ] Check that files are cleaned up or archived after review.

## 📌 เช็คลิสต์ (ภาษาไทย)
- [ ] ตรวจสอบว่าไฟล์ทั้งหมดใน `artifacts/` เป็นผลลัพธ์จาก workflow หรือการรันงานแบบ manual เท่านั้น
- [ ] ยืนยันว่าไม่มี source code สำหรับ production อยู่ใน `artifacts/`
- [ ] ตรวจสอบว่าไฟล์ตั้งชื่อชัดเจน เช่น `ReviewerPolicy.pdf`, `test-report-YYYYMMDD.log`
- [ ] ยืนยันว่า artifacts เป็นไฟล์ชั่วคราว ไม่ปะปนกับ source code
- [ ] ตรวจสอบว่าไฟล์ถูกลบหรือเก็บเข้าคลังหลังรีวิวเสร็จ
