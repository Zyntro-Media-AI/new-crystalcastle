# Code of Conduct

## ✨ Our Commitment

We are committed to providing a welcoming, inclusive, and respectful environment for everyone contributing to this project. This includes contributors, maintainers, users, and reviewers.

We value collaboration, constructive feedback, and technical excellence without discrimination or hostility.

---

## 🤝 Expected Behavior

All participants are expected to:

- Be respectful and considerate in all interactions
- Focus on constructive technical feedback
- Accept and provide feedback professionally
- Respect differing viewpoints and experience levels
- Use inclusive and clear communication

---

## 🚫 Unacceptable Behavior

The following behaviors are not tolerated:

- Harassment, discrimination, or personal attacks
- Toxic or insulting comments in issues, PRs, or discussions
- Spamming, trolling, or disruptive behavior
- Publishing private information without permission
- Deliberate sabotage of code, CI, or workflows

---

## 🧠 Contribution Standards

To maintain code quality and consistency:

- Follow existing project structure and conventions
- Write clear commit messages (prefer Conventional Commits)
- Ensure CI checks pass before requesting review
- Keep pull requests focused and minimal in scope
- Include clear descriptions for changes

---

## ⚙️ Enforcement

Project maintainers have the right to:

- Request changes or improvements
- Close or reject pull requests that violate guidelines
- Temporarily or permanently restrict contributions for repeated violations

All enforcement decisions will be made fairly and with context.

---

## 📬 Reporting Issues

If you experience or witness unacceptable behavior, please report it to maintainers via:

- GitHub Issues (if appropriate)
- Direct maintainer contact (if configured)

All reports will be handled confidentially and respectfully.

---

## 🌱 Philosophy

This project grows through collaboration, experimentation, and respect.

We believe:
> Great software is built by great communities.
